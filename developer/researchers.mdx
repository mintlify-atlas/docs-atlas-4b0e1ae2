---

A **Researcher** performs linguistic analysis on a `Paper` and provides the raw data that assessments use to calculate their scores. Where an `Assessor` decides *what* to measure and produces user-facing feedback, the `Researcher` handles the underlying computation — word counting, sentence detection, keyphrase matching, morphological analysis, and more. Results are computed lazily on first access and can be cached for re-use.

## AbstractResearcher

`AbstractResearcher` is the base class for all researchers. It ships with a full set of language-independent default researches and can be extended with custom research functions.

```javascript
import { AbstractResearcher, Paper } from "yoastseo";

const paper = new Paper( "Text to analyze", {
    keyword: "analyze",
} );

const researcher = new AbstractResearcher( paper );
```

### Key methods

| Method | Description |
|--------|-------------|
| `getResearch( name )` | Run a research by name and return its result. Returns `false` if the research does not exist. |
| `setPaper( paper )` | Swap the `Paper` the researcher operates on without creating a new instance. |
| `addResearch( name, fn )` | Register a custom research function. |
| `addResearchData( name, data )` | Inject pre-computed data for a research (useful for server-side data). |
| `addHelper( name, fn )` | Register a custom helper function that researches can reference. |
| `addConfig( name, value )` | Register a custom configuration value (e.g., language-specific thresholds). |
| `hasResearch( name )` | Check whether a named research exists. |
| `getAvailableResearches()` | Returns the merged object of default and custom researches. |
| `getData( name )` | Retrieve injected research data by name. |
| `getConfig( name )` | Retrieve a configuration value by name. |
| `getHelper( name )` | Retrieve a helper function by name. |

### Importing a language researcher

Language researchers are not included in the main `yoastseo` entry point. Import them directly from the build path:

```javascript
import EnglishResearcher from "yoastseo/build/languageProcessing/languages/en/Researcher";

const researcher = new EnglishResearcher( paper );
```

## Running research

Call `getResearch()` with the name of the research you need. The researcher executes the corresponding function with the current `Paper` and returns the result.

```javascript
import { AbstractResearcher, Paper } from "yoastseo";

const paper = new Paper( "The quick brown fox jumps over the lazy dog.", {
    keyword: "fox",
} );
const researcher = new AbstractResearcher( paper );

const wordCount    = researcher.getResearch( "wordCountInText" );
const sentences    = researcher.getResearch( "sentences" );
const keyphraseCount = researcher.getResearch( "getKeyphraseCount" );

console.log( wordCount );      // 9
console.log( sentences );      // [ "The quick brown fox jumps over the lazy dog." ]
console.log( keyphraseCount ); // { count: 1, ... }
```

## Relationship between Paper, Researcher, and Assessor

The three core objects work together in a well-defined pipeline:

```
Paper  ──────────────────────────────┐
  │                                  │
  ▼                                  ▼
Researcher  ──── research data ────► Assessor
  │                                  │
  │                                  ▼
  └──────────────────────────────► Assessment
                                     │
                                     ▼
                                  AssessmentResult
```

<Steps>
  <Step title="Create a Paper">
    Instantiate a `Paper` with the text and metadata you want to analyze.
  </Step>
  <Step title="Create a Researcher">
    Pass the `Paper` to a language-specific (or `Abstract`) `Researcher`. The researcher holds a reference to the paper and exposes all research methods.
  </Step>
  <Step title="Create an Assessor">
    Pass the `Researcher` to an `Assessor` such as `SeoAssessor` or `ContentAssessor`. The assessor holds a list of `Assessment` instances.
  </Step>
  <Step title="Run assessments">
    Call `assessor.assess( paper )`. The assessor calls `researcher.setPaper( paper )` internally, then runs each applicable assessment. Each assessment calls `researcher.getResearch()` to obtain the data it needs.
  </Step>
  <Step title="Read results">
    Call `assessor.getValidResults()` to retrieve the array of `AssessmentResult` objects.
  </Step>
</Steps>

## Adding custom research

Register additional research functions on any researcher instance. Custom research functions receive the `Paper` and the `Researcher` as arguments.

```javascript
researcher.addResearch( "myWordFrequency", ( paper, researcher ) => {
    const text = paper.getText();
    const words = text.toLowerCase().match( /\b\w+\b/g ) || [];
    const frequency = {};
    words.forEach( word => {
        frequency[ word ] = ( frequency[ word ] || 0 ) + 1;
    } );
    return frequency;
} );

const frequency = researcher.getResearch( "myWordFrequency" );
```

<Warning>
  Custom research names must be non-empty strings. Passing an empty string or a non-function callback throws a `MissingArgument` or `InvalidTypeError` respectively.
</Warning>

## Injecting research data

For data that is expensive to compute or must come from an external source (for example, server-side morphology data), use `addResearchData()` to inject the data directly and retrieve it with `getData()`:

```javascript
researcher.addResearchData( "morphologyData", {
    en: { /* morphology data object */ },
} );

const morphData = researcher.getData( "morphologyData" );
```

This pattern is used internally by the Yoast SEO WordPress plugin to pass morphology data files to the web worker without recomputing them.
